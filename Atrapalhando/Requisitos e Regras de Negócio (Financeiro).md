## Requisitos Funcionais

### 💰 Contas Hospitalares

- RF01 - O sistema deve gerar uma conta hospitalar a partir de uma internação
- RF02 - O sistema deve vincular a conta hospitalar ao paciente
- RF03 - O sistema deve vincular a conta hospitalar ao convênio
- RF04 - O sistema deve calcular permanência hospitalar com base na internação e alta
- RF05 - O sistema deve calcular diárias de leitos

---

### 🛏️ Leitos e Diárias

- RF06 - O sistema deve calcular valores de diárias por leito/quarto
- RF07 - O sistema deve registrar ocupação de leitos para faturamento
- RF08 - O sistema deve considerar tipo de quarto no cálculo financeiro

---

### 📄 Guias e Convênios

- RF09 - O sistema deve gerar guias de faturamento para convênios
- RF10 - O sistema deve permitir consultar guias por convênio
- RF11 - O sistema deve validar situação do convênio no faturamento
- RF12 - O sistema deve registrar número de autorização da internação
- RF13 - O sistema deve registrar número da carteirinha do convênio

---

### 🧾 Altas

- RF14 - O sistema deve permitir lançar alta hospitalar
- RF15 - O sistema deve encerrar a conta hospitalar após a alta
- RF16 - O sistema deve liberar automaticamente o leito após alta
- RF17 - O sistema deve permitir consultar altas por período
- RF18 - O sistema deve gerar relatório de altas

---

### 👨‍⚕️ Honorários Médicos

- RF19 - O sistema deve vincular médico responsável à conta hospitalar
- RF20 - O sistema deve calcular honorários médicos
- RF21 - O sistema deve permitir consultar faturamento por médico

---

### 📊 Relatórios Financeiros

- RF22 - O sistema deve gerar relatório de internações faturadas
- RF23 - O sistema deve gerar relatório financeiro por convênio
- RF24 - O sistema deve gerar relatório de ocupação de leitos
- RF25 - O sistema deve gerar relatório de pacientes internados
- RF26 - O sistema deve gerar relatório de permanência hospitalar

---

### 🚑 Emergência/Urgência

- RF27 - O sistema deve registrar atendimentos de emergência para faturamento
- RF28 - O sistema deve permitir gerar cobrança de atendimentos emergenciais
- RF29 - O sistema deve permitir consultar atendimentos emergenciais por período

---

### 🔍 Conferência

- RF30 - O sistema deve permitir conferência de contas hospitalares
- RF31 - O sistema deve permitir consultar contas pendentes de faturamento
- RF32 - O sistema deve permitir consultar contas fechadas

---

- RF33 - O sistema deve permitir cadastrar serviços hospitalares
- RF34 - O sistema deve classificar serviços como diária ou taxa
- RF35 - O sistema deve vincular serviços a internações
- RF36 - O sistema deve calcular valores de diárias automaticamente
- RF37 - O sistema deve calcular taxas hospitalares
- RF38 - O sistema deve gerar cobrança de serviços hospitalares
- RF39 - O sistema deve permitir consultar serviços utilizados por internação
- RF40 - O sistema deve permitir cadastrar tabelas de preços por convênio
- RF41 - O sistema deve permitir definir valores de serviços hospitalares por convênio
- RF42 - O sistema deve considerar a tabela do convênio no faturamento
- RF43 - O sistema deve permitir consultar valores de serviços por convênio
- RF44 - O sistema deve permitir controlar vigência de tabelas de preços
